Let's recall that the MITRE ATT&CK framework outlines various tactics and techniques that attackers use. The tactics are categorized into different phases of an attack.

Starting from the beginning:

1. **Initial Access**: This is how the attacker gets into the system in the first place. Techniques here might include phishing, exploiting vulnerabilities, etc.

2. **Execution**: Once initial access is gained, the attacker needs to execute code on the target system.

3. **Persistence**: The attacker wants to maintain access over time, so they might create backdoors or modify system settings to ensure they can return.

4. **Privilege Escalation**: If necessary, the attacker may try to gain higher privileges to access more sensitive parts of the system.

5. **Defense Evasion**: Attackers use techniques to avoid detection by security solutions.

6. **Credential Access**: Stealing account names and passwords, API keys, etc.

7. **Discovery**: The attacker gathers information about the system and network to plan further actions.

8. **Lateral Movement**: Moving across the network to access other systems.

9. **Collection**: Gathering data that is the target of the attack.

10. **Exfiltration**: Sending the collected data out of the network to the attacker's control.

11. **Impact**: Actions that affect the confidentiality, integrity, or availability of the target system, like data deletion or encryption.
