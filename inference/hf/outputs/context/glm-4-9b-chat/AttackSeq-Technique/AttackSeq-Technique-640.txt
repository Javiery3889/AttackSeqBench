
Step 1: Analyze the CTI Outline
- Reconnaissance: The outline mentions the use of custom-built backdoors and RATs, as well as publicly available RATs like PoisonIvy, and a proxy tool named HTRAN.
- Resource Development: Similar to the Reconnaissance phase, this phase also discusses the use of custom-built backdoors and RATs, and the proxy tool HTRAN.
- Initial Access: Spear-phishing emails with executable files embedded in ZIP archives or password-protected documents are used.
- Execution: The use of evasion techniques such as environment-based evasion, large file sizes, and password-protected documents is described. The 'CPU core check' technique is used by the first-stage payload 'CT/NewCT' to detect virtualized environments.
- Persistence: The CT/NewCT implant uses registry entries for persistence.
- Defense Evasion: The use of environment-based evasion, large file sizes, and password-protected documents is mentioned again, along with the 'CPU core check' technique for virtualized environments.
- Others: The geographical locations and targets of the Moafee and DragonOK groups are discussed.
