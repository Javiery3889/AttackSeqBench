### Step-by-Step Analysis

#### 1. Analyze the CTI Report

**Initial Access:**
- Weaponized RTF file exploiting RoyalRoad tool to drop 'intel.wll' loader.
- Archive files with legitimate executables and malicious DLLs for DLL hijacking.
- Direct delivery via an executable file serving as a loader.

**Execution:**
- Aria-body loader establishes persistence, injects itself into processes, decrypts configuration data, contacts C&C servers, and loads the Aria-body backdoor.

**Persistence:**
- Established through 'Startup' folder or 'Run' registry key.
- Injected into processes like 'rundll32.exe' and 'dllhost.exe'.

**Defense Evasion:**
- C&C servers operated within limited daily windows.
- Used fileless loading techniques.
- Frequently changed server infrastructure and loader variants.

**Collection:**
- Capabilities include creating/deleting files and directories, taking screenshots, searching for files, launching files, enumerating process loaded modules, gathering file metadata, closing TCP sessions, collecting OS information, and verifying location using external services.

**Command and Control:**
- Used HTTP and TCP protocols for C&C communication.
- Sent collected data to the C&C domain with an XORed password.
- Backdoor kept listening for additional commands from the server.

**Others:**
- Targeted national government entities in the Asia Pacific region.
- Expanded footholds by attacking from one compromised entity to another.
- Used infected ministries as C&C servers.
- Infrastructure primarily used GoDaddy for domain registration and Alibaba for hosting.
