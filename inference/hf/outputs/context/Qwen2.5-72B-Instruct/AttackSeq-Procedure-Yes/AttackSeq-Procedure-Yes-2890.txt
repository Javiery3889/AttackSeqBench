### Step-by-Step Analysis

#### 1. Analyze the CTI Report

**Initial Access:**
- The 'yty' framework uses a malicious Excel document named 'Circular.xls' with macros to execute an embedded executable.

**Execution:**
- The framework includes two downloaders: a stripped-down C++ program (Downloader 1) and a more functional downloader (Downloader 2, 'Setup.exe').
- 'Setup.exe' checks for a single instance and includes anti-sandbox detection.

**Persistence:**
- A secondary macro in 'Circular.xls' establishes persistence for 'Setup.exe'.

**Defense Evasion:**
- The framework uses junk code and anti-sandbox detection to avoid analysis.
- It creates logs for debugging purposes.

**Command and Control:**
- The framework uses a football-themed communication strategy to disguise traffic.
- It downloads a file from Google Docs to get the C2 host and sends beacon messages containing system information.
- The C2 server responds with instructions for further actions.

**Exfiltration:**
- The framework exfiltrates data such as file listings, keystrokes, and screenshots back to the C2 server using plugins like 'abode.exe'.

**Others:**
- The Donot Team is believed to be behind the 'yty' framework.
- They mimic legitimate applications and services and use domains with slight alterations from legitimate ones.
- The TTPs, infrastructure, and malware code are similar to the EHDevel malware, suggesting the same group of operators.
- The evolution from EHDevel to 'yty' indicates continuous improvement.
