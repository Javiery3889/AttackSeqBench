### Step-by-Step Analysis

#### 1. Analyze the CTI Report

**Initial Access:**
- Weaponized RTF file exploiting RoyalRoad tool to drop 'intel.wll' loader.
- Archive files with a legitimate executable and a malicious DLL for DLL hijacking.
- Direct delivery via an executable file serving as a loader.

**Execution:**
- Aria-body loader establishes persistence, injects itself into processes, decrypts configuration data, contacts C&C servers, and loads the Aria-body backdoor.

**Persistence:**
- Established through the 'Startup' folder or 'Run' registry key.
- Injected into processes like 'rundll32.exe' and 'dllhost.exe'.

**Defense Evasion:**
- C&C servers operate within limited daily windows.
- Fileless loading techniques.
- Frequent changes in server infrastructure and loader variants.

**Discovery:**
- Aria-body backdoor gathers detailed information from the victim's machine, including host-name, computer-name, username, domain name, windows version, processor speed, MachineGuid, system architecture, public IP, installed software, active processes, running windows, TCP and UDP table status, and services data.

**Collection:**
- Capabilities to create/delete files and directories, take screenshots, search for files, launch files, enumerate process loaded modules, gather file metadata, close TCP sessions, collect OS information, and verify location using external services.

**Others:**
- Targeted national government entities in the Asia Pacific region.
- Expanded footholds by launching attacks from one compromised government entity to another.
- Used infected ministries as C&C servers.
- Infrastructure primarily used GoDaddy for domain registration and Alibaba for hosting.
- Attribution based on similarities in targeting, strings, hashing functions, code, and infrastructure overlap with previous operations.

**Question:**

**Answer Choices:**

#### 3. Compare and Evaluate

**Key Points from the CTI Report:**
- The Aria-body backdoor gathers detailed information from the victim's machine.
- The backdoor has capabilities to execute additional commands.
- The backdoor uses HTTP and TCP protocols for C&C communication.

**Reasoning:**
- The Aria-body backdoor is designed to gather detailed information from the victim's machine.
- Once the information is collected, the backdoor can receive additional commands from the C&C server.
- The C&C communication is established using HTTP and TCP protocols.
- The report indicates that the backdoor has the capability to execute additional commands, which suggests that it is likely to do so after collecting information.
