### Step-by-Step Analysis

#### 1. Analyze the CTI Report

**Initial Access:**
- The APT-TOCS attack utilized "powershell.exe" to execute Shellcode scripts for remote control of targeted systems.
- Security vulnerabilities were used to execute scripts directly on the host.
- A binary file named Sample A (Trojan/Win32.MSShell) was used to execute embedded malware using WinExec to call "powershell.exe".

**Execution:**
- The core step involved downloading Shellcode script functionalities and executing them in memory without writing to the hard disk.
- Communication via Beacon mode, sending network heartbeat packages every 60 seconds, and transmitting data information via the Cookie field.

**Defense Evasion:**
- Features designed to evade detection by security software and firewall interception.
- Running in memory, no hard disk writing operations, and multi-beacon communication to evade sandbox detection.

**Lateral Movement:**
- APT-TOCS can laterally move within the internal network as required to conduct an attack.

**Others:**
- APT-TOCS is closely related to the Cobalt Strike platform.
- Capable of penetrating firewalls, undetectable, and can attack various platforms.
- Active for 5 years with strong anti-detection and concealment capabilities.
- Considered an APT due to its targeted nature and use of a commercial attack platform.
