### Step-by-Step Analysis

#### 1. Analyze the CTI Report

**Initial Access:**
- UEFI firmware-level compromise.
- MoonBounce implant embedded in the SPI flash.
- Malicious code introduced into the boot sequence.

**Execution:**
- MoonBounce incorporated into the CORE_DXE component.
- Hooks set to intercept functions like AllocatePool, CreateEventEx, and ExitBootServices.
- Propagation of malicious code to the Windows kernel's memory address space.

**Persistence:**
- MoonBounce persists on the SPI flash.
- Survives disk formatting or replacement.

**Defense Evasion:**
- Operates in memory only.
- Fileless attack with a small footprint.
- Modifies existing firmware component rather than adding a new one.

**Command and Control:**
- User-mode malware reaches out to a hardcoded C&C URL.
- Fetches another stage of the payload to run in memory.

**Exfiltration:**
- Enumerates hosts and gathers network information.
- Copies files across SMB shares.
- Dumps the Active Directory domain database.
- Uses tools like Sysinternals Psexec and WMI for remote command execution.

**Others:**
- Attributed to APT41.
- Highly targeted attack.
- Maintains infrastructure from at least March 2020.
- Vendors taking precautions against UEFI threats.
