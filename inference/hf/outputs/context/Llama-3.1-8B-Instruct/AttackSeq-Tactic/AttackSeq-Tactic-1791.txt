**Step 1: Analyze the CTI Outline**

The CTI outline describes the attack sequence of the Metamorfo campaigns as follows:

1. Initial Access: Phishing emails with HTML attachments redirect victims to cloud storage sites to download a ZIP file containing a malicious executable.
2. Execution: The malicious executable is run, continuing the infection chain.
3. Persistence: Registry keys are created in Software\Microsoft\Windows\CurrentVersion\Run, and a VBS script is set to persist and download the same ZIP archive.
4. Defense Evasion: Techniques are used to evade detection, including public cloud infrastructure and legitimate signed binaries.
5. Discovery: The banking Trojan searches for antivirus and banking software folders, collects hostname and OS version information, and sends it to a hardcoded domain.
6. Collection: Screenshots and mouse clicks are captured while the user browses banking sites.
7. Command and Control: The C2 server is selected based on a string in a file named 'id', and the malware communicates with the C2 server over raw TCP on port 9999.
