The CTI outline describes the attack sequence of the Dark Seoul attackers, also known as Operation Troy, in the following order:

1. Reconnaissance: The attackers used malware to perform basic reconnaissance on the infected host and return the results to the attacker.
2. Initial Access: The attackers used a spear-phishing email with a trojanized version of a legitimate software installation executable to gain initial access.
3. Execution: The malware, TDrop2, executed a legitimate Microsoft Windows executable from the system32 folder and used process hollowing to inject malicious code into the process.
4. Defense Evasion: The TDrop2 malware used process hollowing and obfuscated its second-stage payload retrieval by disguising it as an image file.
5. Command and Control: The malware communicated with C2 servers using encrypted and encoded commands.
